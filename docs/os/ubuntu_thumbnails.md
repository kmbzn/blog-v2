# 우분투 이미지 비디오 썸네일(미리보기) 안 보임 문제 해결

우분투 (특히 24.04 LTS)를 사용하다 보면 파일 탐색기 상에서 이미지나 비디오 파일의 썸네일이 어느 순간 사라지거나, 아예 생성되지 않는 현상을 겪게 됩니다. Reddit 등 커뮤니티에 따르면 25.10 이후 버전에서는 해결되었다고 하지만, 안정성을 중시하는 사용자들은 여전히 24.04 LTS를 고수할 필요가 있습니다. 현재 26.04 LTS가 출시되었음에도 불구하고 초기 불안정성 때문에 24.04 LTS 버전을 사용하는 저같은 분들을 위해, 이 고질적인 문제를 해결하는 확실한 방법을 공유합니다.

## 1. 비디오 썸네일러 설치 (optional)

가장 먼저 비디오 파일의 썸네일을 생성해 줄 도구를 설치해야 합니다. 우분투 기본 상태에서는 비디오 썸네일 생성 기능이 없으므로 `ffmpegthumbnailer`를 먼저 설치해 줍니다. 이미 설치했거나 비디오의 썸네일이 필요 없는 경우 skip해도 됩니다.

```sh
sudo apt update
sudo apt install ffmpegthumbnailer
```

## 2. 핵심 원인: AppArmor 보안 설정 해제

비디오 썸네일러를 설치했더라도 썸네일이 여전히 표시되지 않을 수 있습니다. 우분투 24.04의 보안 정책인 AppArmor가 썸네일을 만드는 '샌드박스(bwrap)' 프로세스를 차단하고 있기 때문입니다.

한 번 차단당한 파일은 시스템에서 실패된 것으로 간주하여 다시는 생성을 시도하지 않게 됩니다. 따라서 아래 명령어를 통해 비특권 유저 네임스페이스 제한을 풀어주어야 합니다.

### 커널 설정 변경 (즉시 적용)

```sh
sudo sysctl -w kernel.apparmor_restrict_unprivileged_userns=0
```

### 재부팅 후에도 유지되도록 설정 저장

```sh
echo "kernel.apparmor_restrict_unprivileged_userns = 0" | sudo tee /etc/sysctl.d/60-apparmor-thumbnail.conf
```

## 3. 실패한 썸네일 캐시 강제 삭제

설정을 바꿨다고 해서 바로 썸네일이 표시되지 않을 수 있습니다. 이미 시스템 내부적으로 "이 파일들은 썸네일 생성에 실패한 파일들"이라는 낙인이 찍혀 있기 때문입니다. 이 기록을 지워주어야 파일 관리자가 다시 생성을 시도합니다.

### 캐시 삭제 및 파일 관리자 재시작

```sh
# 실패한 기록이 담긴 캐시 폴더 삭제
rm -rf ~/.cache/thumbnails/fail/gnome-thumbnail-factory/*

# 파일 관리자(Nautilus) 완전히 종료 후 재시작
nautilus -q
```

## Final Check

위 과정을 모두 마치고 다시 파일 탐색기에서 폴더를 열어보면, (그동안 보이지 않던) JPG, PNG 이미지와 MP4 등 비디오 파일들의 썸네일이 표시되는 것을 확인할 수 있습니다.
