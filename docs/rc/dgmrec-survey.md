# DGMRec

## MRS의 장점
- CF 정보만으로는 정확한 추천이 어려우니, modality 정보의 도움을 받아서 더 확실한 추천을 할 수 있다.

## 기존 MRS의 한계점
1. Modality 결측이 없는 상황을 가정한 모델이지만 실제 상황에서는 결측 상황이 있다.
  - 결측 아이템을 제외시키거나, 평균값 주입 같은 방식을 쓰지만 성능의 저하가 있다.
2. 같은 아이템의 서로 다른 modality가 동일한 의미를 갖는다고 가정하여 직접 alignment함.
  - 이미지는 시각적 속성을, 텍스트는 설명적 맥락 담는 등 각 modality마다 갖는 고유한 정보가 있으므로 이 가정이 항상 성립하지는 않습니다.

### [5문단] (C1 분석)
- NN 주입 방식이 실패하는 원인을 LGMRec으로 분석한 결과, 주입된 특징이 원래 modality를 제대로 대체하지 못하는 것이 성능 저하의 원인이었습니다.
- 최신 MRS일수록 Modality feature의 영향을 많이 받기 때문에 그 저하의 정도가 더 심해집니다.

### [11문단] (C2 해결)

### [12문단]
- 두 과제는 밀접하게 연결되어 있으며, modality 속성의 분리 품질이 결측 특징 생성의 품질을 직접 좌우합니다.

### [13문단] (기여 요약)

## 3. Methodology

### [도입 1문단]
- 결측 modality를 정확하게 생성하려면 일반 특징과 특수 특징을 모두 고려해야 하므로, 별도 인코더로 두 특징을 추출하는 Disentangling Modality Feature 모듈을 제안하고 분리와 alignment을 위한 정보 기반 손실을 도입합니다.

### [도입 2문단]
- 결측 상황에서 의미 있는 표현을 얻고 item-item 그래프를 구성하기 위해, 일반 및 특수 특징을 생성하고 이를 사용해 그래프를 적응적으로 정제하는 Missing Modality Generation 모듈을 제안합니다.

### [도입 3문단]
- Modality 특징은 의미 정보에는 강하지만 추천에 필요한 협업 지식이 부족하므로, 사용자와 아이템 표현을 잇는 두 가지 alignment 방법을 추가로 도입합니다.

### [3.1 Preliminaries]
- 사용자 집합, 아이템 집합, 상호작용 행렬 R을 정의하고, 각 아이템의 modality 특징 X_m은 사전 학습 모델(이미지는 CNN, 텍스트는 SBERT)로 추출하며 결측 특징은 평균값으로 초기화합니다.

### [3.2 도입]
- 기존 MRS는 직접 alignment이나 인접 뷰 결합으로 modality의 고유 특성을 흐리므로, 인코더로 일반 특징과 특수 특징을 나눠 추출하고 GCN과 정보 기반 분리로 이를 정제합니다.

### [3.2.1 - 1문단]
- Modality 전반에 파라미터를 공유하는 일반 인코더와 modality별 독립 파라미터를 갖는 특수 인코더, 두 개의 별도 인코더로 각 특징을 추출합니다.

### [3.2.1 - 2문단]
- 일반 인코더는 공통 속성을, 특수 인코더는 각 modality의 고유 특성을 포착하며, modality마다 차원이 다른 X_m을 통일된 차원으로 투영하는 레이어 h_m을 별도로 둡니다.

### [3.2.1 - 3문단]
- 사용자 modality 선호 임베딩을 도입해 아이템 선호 행렬을 계산하고, 이를 통해 modality 특징을 사용자 선호와 alignment하여 추천과 무관한 잡음이 제거된 특징을 얻습니다.

### [3.2.1 - 4문단]
- 잡음이 제거된 특징을 modality 유사도 기반 item-item 그래프 위에서 LightGCN으로 강화하고, 사용자가 상호작용한 아이템의 특징을 집계해 사용자 modality 특징을 계산합니다.

### [3.2.2 - 1문단]
- 인코더 분리만으로는 효과적인 분리가 보장되지 않으므로, 대조 손실 두 가지를 활용하는 정보 기반 접근을 도입합니다.

### [3.2.2 - 2문단]
- 같은 modality 안에서 일반 특징과 특수 특징 간 상호 정보를 최소화하기 위해 CLUB을 적용하며, 이를 반복적으로 최소화해 두 특징이 상호 보완적인 정보를 갖도록 유도합니다.

### [3.2.2 - 3문단]
- 서로 다른 modality의 일반 특징 간 상호 정보를 최대화하기 위해 InfoNCE를 채택하여 일반 특징들을 같은 잠재 공간으로 alignment합니다.

### [3.2.2 - 4문단]
- 두 손실의 합인 분리 손실을 통해 modality 내 분리와 modality 간 일반 특징 alignment을 동시에 달성하며, 이것이 결측 표현 생성을 위한 cross-modality 정보 활용의 기반이 됩니다.

### [3.3 도입]
- 분리된 특징을 바탕으로 원시 특징을 재구성하는 디코더를 학습하고, 결측 아이템의 일반 및 특수 특징을 생성하며, 생성 결과로 item-item 그래프를 정제합니다.

### [3.3.1 Reconstruction]
- 일반 특징과 특수 특징으로부터 원시 특징 X_m을 복원하는 디코더를 재구성 손실로 학습하여, 분리된 특징이 무의미한 표현이 아니라 원시 특징을 복원할 수 있는 의미 있는 정보를 유지하도록 보장합니다.

### [3.3.2 - 1문단]
- 일반 특징과 특수 특징은 성질이 근본적으로 다르므로 생성에도 별개의 접근이 필요합니다.

### [3.3.2 - 2문단]
- 일반 특징은 InfoNCE로 alignment된 다른 modality의 일반 특징을 2층 MLP 생성기에 넣어 만들며, 입력 modality도 결측이면 특징 평균을 사용해 입력 차원을 일정하게 유지합니다.

### [3.3.2 - 3문단]
- 특수 특징은 다른 modality에서 가져올 수 없으므로, 상호작용한 사용자들의 modality 선호에 암묵적으로 담긴 아이템 정보를 별도 생성기로 활용해 만듭니다.

### [3.3.2 - 4문단]
- 생성된 특징이 원래 정보를 보존하도록 원래 특징과 생성 특징을 유사하게 만드는 생성 손실을 도입하며, 재구성 및 생성 손실은 modality가 있는 아이템에 대해서만 계산합니다.

### [3.3.3 - 1문단]
- 일정 epoch 간격으로 결측 아이템의 특징을 생성하고 디코더로 원시 특징을 근사하되, 단순 대체만으로는 부족하므로 생성 특징 기반의 새로운 인접 행렬을 구성해 그래프를 갱신합니다.

### [3.3.3 - 2문단]
- 급격한 그래프 변화로 인한 학습 불안정을 막기 위해 하이퍼파라미터 alpha로 새 연결을 부드럽게 통합하는 적응적 업데이트 전략을 사용합니다.

### [3.3.3 - 3문단]
- 결측 아이템과 연관된 엣지만, 그것도 modality 보유 아이템에서 결측 아이템으로 향하는 방향성 엣지로만 갱신하여, 미숙한 생성 특징에 의한 원래 특징의 오염을 막고 유사도 계산 비용도 줄입니다.

### [3.3.3 - 선행 연구 비교]
- 인접 행렬을 추천 손실과 공동 최적화하는 LATTICE, MICRO와 달리, 디코더로 원시 특징을 정확히 재구성한 뒤 이를 사용해 인접 행렬을 만들기 때문에 결측 아이템의 엣지 연결이 가능합니다.

### [3.4 도입]
- Modality 특징의 협업 지식 부족을 해결하기 위해, 사전 학습 특징을 ID 임베딩과 alignment해 modality 지식과 협업 지식을 통합하는 두 가지 alignment 방법을 제안합니다.

### [3.4.1 Fusing]
- 모든 modality의 일반 특징을 평균 풀링하고 이를 특수 특징과 다시 풀링해 최종 modality 표현을 만든 뒤, ID 임베딩과 결합해 최종 표현을 얻고 내적으로 추천 점수를 계산합니다.

### [3.4.2 Behavior-Modality Alignment]
- 사용자 행동과 modality 특징을 대조 손실로 alignment해 modality 특징에 협업 지식을 주입합니다.

### [3.4.3 User-Item Alignment]
- 아이템의 modality 특징을 상호작용한 사용자의 modality 특징과 alignment해 각 modality 내에서의 일관성을 높입니다.

### [최종 목적 함수]
- BPR 손실에 재구성, 생성, 분리, alignment 손실을 더한 것이 최종 목적 함수이며, 분리와 alignment 손실의 비중은 하이퍼파라미터 $\lambda_1$, $\lambda_2$로 조절합니다.

## [3.1 Preliminaries]

- 구현 관점에서 준비물은 세 가지입니다. 상호작용 행렬 $R$ (보통 sparse matrix), 사전 추출된 modality 특징 $X_m$ (이미지면 $|I| \times 4096$, 텍스트면 $|I| \times 384$ 같은 numpy 배열을 로드), 그리고 `nn.Embedding`으로 만드는 ID 임베딩 $E_{id}$입니다. 결측 아이템의 행은 $X_m$의 평균 벡터로 채워서 시작합니다. 즉 `X_m[missing_idx] = X_m[available_idx].mean(0)` 한 줄이 초기화의 전부입니다.

## [3.2.1 인코딩, Eq 1]

- 수식은 $E^g_m = f^g(h_m(X_m))$, $E^s_m = f^s_m(X_m)$입니다. 코드로 보면 $h_m$은 `nn.Linear`($d_m$, 64)를 modality마다 하나씩, $f^g$는 nn.Linear(64, 64) 딱 하나를 만들어서 이미지 경로에도 텍스트 경로에도 같은 객체를 재사용합니다. 파라미터 공유라는 게 별게 아니라 같은 레이어 인스턴스를 두 번 호출하는 것입니다. 반면 $f^s_m$은 nn.Linear($d_m$, 64)를 modality 개수만큼 따로 만듭니다. 그림 2 왼쪽에서 $X_v$와 $X_t$가 $f^g$ 하나로 모이고 $f^s_v$, $f^s_t$로는 갈라지는 부분이 이겁니다.

## [3.2.1 선호 임베딩, Eq 2와 Eq 3]

- Eq 2는 $p_{i,m} = \frac{1}{|N_i|} \sum_{u \in N_i} p_{u,m}$인데, 아이템 $i$와 상호작용한 유저들의 선호 임베딩 평균입니다. 코드로는 정규화된 상호작용 행렬을 선호 임베딩 행렬에 곱하는 것, 즉 `P_i = R_norm.T @ P_u` 한 번의 sparse matmul로 끝납니다. Eq 3의 $\tilde{E}_m = E_m \odot \sigma(P_{i,m})$이 재미있는 부분인데, sigmoid를 통과시킨 선호 벡터를 원소별 곱으로 씌웁니다. sigmoid 출력이 0에서 1 사이니까 이건 사실상 차원별 게이트입니다. 유저 선호와 관련 없는 차원은 0 쪽으로 눌러서 꺼버리고 관련 있는 차원만 통과시키는 소프트 마스크라고 보면 됩니다. "노이즈 제거"의 실체가 이 게이팅입니다.

## [3.2.1 그래프 전파, Eq 4와 Eq 5]

- Eq 4는 그냥 코사인 유사도입니다. $S^m_{i,j} = \frac{(x_{i,m})^\top x_{j,m}}{\|x_{i,m}\| \|x_{j,m}\|}$이니까 $X_m$을 행 단위로 L2 정규화한 뒤 `X_norm @ X_norm.T`를 구하고, 행마다 top-k만 남기고 나머지는 0으로 만들어 sparse화한 것이 $S^m$입니다. Eq 5는 LightGCN인데 수식이 $\bar{E}^{(l)}_m = S^m \cdot \bar{E}^{(l-1)}_m$이 전부입니다. 가중치도 활성함수도 없이 인접 행렬을 $L$번 곱하는 것뿐이라, for 루프 안에서 `E = torch.sparse.mm(S, E)`를 $L$번 도는 코드가 됩니다. 마지막 층 출력 $\bar{E}_m = \bar{E}^{(L)}_m$만 씁니다. Eq 6의 유저 특징 $\bar{f}_{u,m} = \frac{1}{|N_u|} \sum_{i \in N_u} \bar{e}_{i,m}$도 Eq 2와 같은 패턴으로, 유저가 상호작용한 아이템 특징의 평균이니 `R_norm @ E_m`입니다.

## [3.2.2 CLUB, Eq 7]

- 여기가 구현이 제일 까다로운 부분입니다. $q_\varphi$는 특수 특징을 받아서 일반 특징의 분포를 예측하는 별도의 2층 MLP 네트워크입니다. 보통 평균과 로그분산을 출력하게 만들어서 가우시안 log likelihood를 계산합니다. Eq 7을 보면 첫 항은 짝이 맞는 샘플의 $\log q_\varphi(\bar{e}^g_{i,m} | \bar{e}^s_{i,m})$이고 둘째 항은 짝이 안 맞는 $(i, j)$ 조합의 $\frac{1}{|I|} \sum_j \log q_\varphi(\bar{e}^g_{j,m} | \bar{e}^s_{i,m})$ 평균입니다. 직관적으로는 "내 특수 특징으로 내 일반 특징은 잘 예측되는데 남의 일반 특징도 똑같이 잘 예측된다면 둘 사이에 정보가 없는 것"이라는 논리로 상호 정보 상한을 추정합니다. 실전에서는 학습이 두 갈래로 번갈아 돌아갑니다. $q_\varphi$ 자체는 조건부 분포를 잘 맞추도록 likelihood를 최대화하는 방향으로 따로 업데이트하고, 본체 인코더는 그 $q_\varphi$로 계산한 $L_{club}$을 낮추는 방향으로 업데이트합니다. 논문에서 "iteratively minimize alongside other parameters"라고 쓴 게 이 교대 최적화를 뜻합니다.

## [3.2.2 InfoNCE, Eq 8]

- 이건 표준 contrastive loss라 구현이 간단합니다. $L_{InfoNCE} = \sum_{i \in I} -\log \frac{\exp(\bar{e}^g_{i,m} \cdot \bar{e}^g_{i,m'})}{\sum_{j \in I} \exp(\bar{e}^g_{i,m} \cdot \bar{e}^g_{j,m'})}$에서 같은 아이템 $i$의 이미지 일반 특징과 텍스트 일반 특징의 내적이 분자, 배치 내 모든 아이템 $j$와의 내적이 분모에 들어갑니다. 코드로는 `logits = E_g_img @ E_g_txt.T`를 만들고 대각선이 정답이 되도록 `F.cross_entropy(logits, torch.arange(B))` 하는 전형적인 CLIP 스타일입니다. Eq 9는 $L_{disentangle} = L_{club} + L_{InfoNCE}$로 이 둘을 그냥 더한 것입니다.

## [3.3.1 재구성, Eq 10과 Eq 11]

- 디코더 $f^{dec}_m$은 nn.Linear(128, $d_m$)입니다. 입력이 128인 이유는 Eq 10의 $\bar{X}_m = f^{dec}_m(\bar{E}^g_m \oplus \bar{E}^s_m)$에서 $\oplus$가 concat이라서 64차원 일반 특징과 64차원 특수 특징을 이어 붙이기 때문입니다. 출력 차원이 $d_m$, 즉 원시 특징 차원(4096이나 384)으로 되돌아가는 게 포인트입니다. 손실은 Eq 11의 $L_{recon} = \sum_i MSE(x_{i,m}, \bar{x}_{i,m})$ 그대로 `F.mse_loss(X_recon, X_m)`이고, 결측 아이템은 마스크로 걸러서 손실 계산에서 뺍니다.

## [3.3.2 생성기, Eq 12와 Eq 13]

- 일반 특징 생성기 Eq 12는 $\hat{E}^g_m = G^g_m(\bigoplus_{m'} \bar{E}^g_{m'})$으로, 다른 modality들의 일반 특징을 전부 concat해서 2층 MLP에 넣습니다. modality가 2개면 입력이 64, 3개면 128이 되는 식입니다. 조건 분기가 하나 필요한데, 참조할 modality도 결측이면 그 자리에 해당 modality 일반 특징의 전체 평균 벡터를 끼워 넣어 입력 차원을 고정합니다. 특수 특징 생성기 Eq 13은 $\hat{E}^s_m = G^s_m(P_{i,m})$으로 입력이 아예 다릅니다. Eq 2에서 만든 아이템 선호 임베딩을 받아 2층 MLP를 통과시킵니다. 콘텐츠가 아니라 상호작용에서 오는 신호라 결측이어도 계산 가능한 경로입니다. Eq 14의 $L_{gen} = \sum_i [MSE(\bar{e}^g_{i,m}, \hat{e}^g_{i,m}) + MSE(\bar{e}^s_{i,m}, \hat{e}^s_{i,m})]$은 생성기 출력이 진짜 인코더 출력과 같아지도록 하는 MSE인데, 이것도 modality가 있는 아이템에서만 계산합니다. 말하자면 정답이 있는 아이템으로 생성기를 지도학습해두고, 결측 아이템에서는 추론만 하는 구조입니다.

## [3.3.3 그래프 정제, Eq 15와 Eq 16]

- 매 5 epoch마다 도는 별도 루틴입니다. 결측 아이템에 대해 생성기 두 개를 돌려 $\hat{E}^g_m$, $\hat{E}^s_m$을 얻고, Eq 15의 $\hat{X}_m = f^{dec}_m(\hat{E}^g_m \oplus \hat{E}^s_m)$처럼 concat해서 학습된 디코더에 넣어 원시 차원의 특징을 만듭니다. 이걸로 Eq 4와 같은 코사인 유사도를 다시 계산해 새 인접 행렬 $\hat{S}^m$을 만드는데, 전체 쌍이 아니라 결측 아이템이 낀 쌍만 계산합니다. Eq 16의 $S^m = \alpha S^m + (1-\alpha)\hat{S}^m$은 기존 그래프와 새 그래프의 지수이동평균 같은 블렌딩입니다. $\alpha = 1$이면 갱신 안 함, $\alpha = 0$이면 통째 교체인데 실험에서 중간값이 최적이었습니다. 엣지 방향도 제약이 있어서, modality 보유 아이템에서 결측 아이템 쪽으로만 향하는 비대칭 행렬을 만듭니다. 코드로 생각하면 $S$의 행 중 결측 아이템에 해당하는 행만 덮어쓰고 열 방향은 건드리지 않는 식입니다. 이래야 아직 덜 학습된 생성 특징이 GCN 전파를 타고 멀쩡한 아이템 표현을 오염시키는 걸 막을 수 있습니다.

## [3.4.1 융합과 점수, Eq 17에서 Eq 19]

- Eq 17은 풀링 두 번입니다. $\bar{E}_M = MeanPool_{m}(\bar{E}^s_m, MeanPool_{m'}(\bar{E}^g_{m'}))$이니까 먼저 modality들의 일반 특징을 평균내고(정렬해놨으니 평균내도 정보가 뭉개지지 않는다는 전제), 그 결과를 각 modality의 특수 특징들과 다시 평균냅니다. Eq 18은 최종 표현이 그냥 덧셈이라는 것, 즉 $\bar{E}_u = E_{u,id} + \bar{F}_M$, $\bar{E}_i = E_{i,id} + \bar{E}_M$입니다. Eq 19의 점수 $y_{u,i} = \bar{e}_u^\top \bar{e}_i$는 내적이니 학습 때는 배치 단위 곱, 추론 때는 `E_u @ E_i.T` 한 방으로 전체 점수 행렬이 나옵니다.

## [3.4.2와 3.4.3 정렬 손실, Eq 20과 Eq 21]

- 둘 다 Eq 8과 같은 InfoNCE 틀인데 무엇을 positive로 잡느냐만 다릅니다. Eq 20의 $L_{BM\text{-}align}$은 같은 유저의 $e_{u,id}$와 $\bar{f}_{u,M}$, 같은 아이템의 $e_{i,id}$와 $\bar{e}_{i,M}$이 짝입니다. CF 쪽 표현과 modality 쪽 표현을 같은 공간으로 끌어오는 역할입니다. Eq 21의 $L_{UI\text{-}align}$은 상호작용이 있었던 $(u, i) \in O$ 쌍에서 유저의 modality 특징 $\bar{f}_{u,m}$과 아이템의 modality 특징 $\bar{e}_{i,m}$이 짝입니다. 구현은 세 경우 모두 내적 행렬 만들고 cross entropy 거는 동일 패턴의 재사용입니다.

## [최종 손실, Eq 23과 Eq 24]

- Eq 23의 BPR은 $L_{bpr} = \sum_{(u,i^+,i^-)} -\sigma(y_{u,i^+} - y_{u,i^-})$로 긍정 아이템 점수에서 부정 샘플 점수를 뺀 값에 sigmoid를 씌우는 표준형입니다. Eq 24의 $L = L_{bpr} + L_{recon} + L_{gen} + \lambda_1 L_{disentangle} + \lambda_2 L_{align}$을 보면 가중치 배치가 눈에 띄는데, $L_{recon}$과 $L_{gen}$은 계수 없이 그대로 더하고 $L_{disentangle}$과 $L_{align}$에만 $\lambda_1$, $\lambda_2$가 붙습니다. 재구성과 생성은 오토인코더가 제대로 서기 위한 필수 조건이라 항상 온전히 걸어두고, 분리와 정렬은 강도를 조절해야 하는 보조 신호로 취급한 설계입니다. 한 학습 스텝을 그려보면 배치 샘플링, forward로 다섯 손실 계산, backward 한 번, 그리고 $q_\varphi$만 따로 한 번 더 업데이트, 여기에 5 epoch마다 그래프 갱신 루틴이 끼어드는 흐름이 됩니다.
